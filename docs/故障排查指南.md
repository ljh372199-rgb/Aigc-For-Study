## 1. 快速问题识别

### 1.1 问题分类矩阵

| 症状 | 可能原因 | 紧急程度 | 排查入口 |
|------|----------|----------|----------|
| 服务无法访问 | 服务宕机/网络问题 | 🔴 紧急 | 健康检查 |
| 页面加载慢 | API 响应慢/数据库慢 | 🟡 高 | API 性能面板 |
| 数据加载失败 | 数据库连接/查询错误 | 🔴 紧急 | 数据库监控 |
| 登录失败 | 认证服务/Redis 问题 | 🔴 紧急 | Redis 监控 |
| 告警风暴 | 配置错误/真实故障 | 🟡 高 | Alertmanager |

### 1.2 快速检查清单

执行以下检查：

```bash
# 1. 检查所有服务状态
docker-compose ps

# 2. 检查资源使用
docker stats --no-stream

# 3. 检查 API 健康
curl http://localhost:8000/health

# 4. 检查数据库连接
docker-compose exec api python -c "from app.core.database import engine; print(engine.connect())"

# 5. 检查 Redis 连接
docker-compose exec redis redis-cli ping

# 6. 检查最近日志
docker-compose logs --tail=50 api | grep -i error
```

### 1.3 监控仪表盘检查路径

当服务出现异常时，按以下顺序检查监控仪表盘：

```
1. 系统概览 (overview)
   ├── 确认告警状态
   ├── 查看服务健康
   └── 确认资源状态

2. API 性能 (aigc-api-performance)
   ├── 查看错误率趋势
   ├── 检查响应时间
   └── 分析慢接口

3. 容器监控 (container-monitoring)
   ├── 确认容器状态
   ├── 检查资源限制
   └── 查看重启次数

4. 系统指标 (system-metrics)
   ├── CPU/内存使用
   ├── 磁盘空间
   └── 网络状态
```

## 2. 常见问题与解决方案

### 2.1 服务启动失败

**症状**: `docker-compose up` 失败

**排查步骤**:

```bash
# 1. 查看详细错误信息
docker-compose up

# 2. 检查端口占用
lsof -i :8000
lsof -i :5432
lsof -i :6379

# 3. 检查 Docker 状态
docker ps -a

# 4. 检查日志
docker-compose logs api
docker-compose logs db
docker-compose logs redis
```

**常见原因与解决方案**:

| 原因 | 解决方案 |
|------|----------|
| 端口被占用 | 停止占用端口的进程或修改 docker-compose.yml |
| 配置文件错误 | 检查 .env 文件格式 |
| 镜像拉取失败 | 检查网络或使用国内镜像源 |
| 磁盘空间不足 | 清理 Docker: `docker system prune -a` |
| 权限问题 | 检查文件权限或使用 sudo |

### 2.2 API 请求超时

**症状**: 前端请求长时间等待后返回 504

**排查步骤**:

```bash
# 1. 检查 API 进程
docker-compose ps api

# 2. 查看 API 日志
docker-compose logs -f api

# 3. 检查响应时间
curl -w "@curl-format.txt" -o /dev/null -s http://localhost:8000/api/v1/courses

# curl-format.txt 内容:
# time_namelookup: %{time_namelookup}\n
# time_connect: %{time_connect}\n
# time_starttransfer: %{time_starttransfer}\n
# time_total: %{time_total}\n
```

**诊断命令**:

```bash
# 检查 API 进程状态
docker-compose exec api ps aux | grep python

# 检查连接数
docker-compose exec api netstat -ant | grep 5432 | wc -l

# 检查数据库慢查询
docker-compose exec db psql -U aigc -c "SELECT * FROM pg_stat_activity WHERE state = 'active' AND query_start < now() - interval '5 seconds';"
```

### 2.3 数据库连接池耗尽

**症状**: API 返回 "connection timeout" 错误

**排查步骤**:

```bash
# 1. 检查当前连接数
docker-compose exec db psql -U aigc -c "SELECT count(*) FROM pg_stat_activity;"

# 2. 检查连接池使用
curl http://localhost:9090/api/v1/query?query=db_connections_active

# 3. 查看等待连接
docker-compose exec db psql -U aigc -c "SELECT * FROM pg_stat_activity WHERE waiting = 't';"

# 4. 检查慢查询
docker-compose exec db psql -U aigc -c "SELECT pid, now() - query_start AS duration, query FROM pg_stat_activity WHERE state = 'active' AND now() - query_start > interval '5 seconds';"
```

**解决方案**:

```sql
-- 终止长时间运行的查询
SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE now() - query_start > interval '30 seconds';

-- 增大连接池 (需要重启)
-- 在 .env 中修改
DATABASE_POOL_SIZE=30
DATABASE_MAX_OVERFLOW=50
```

### 2.4 内存不足 (OOM)

**症状**: 容器被强制终止，dmesg 显示 OOM

**排查步骤**:

```bash
# 1. 检查容器内存使用
docker stats --no-stream

# 2. 检查系统内存
free -h

# 3. 查看 OOM 日志
dmesg | grep -i "out of memory"
journalctl -k | grep -i memory

# 4. 检查容器重启日志
docker inspect api | grep -A 5 "State"
```

**解决方案**:

```bash
# 临时: 重启服务释放内存
docker-compose restart api

# 永久: 增加内存限制或优化代码
# 编辑 docker-compose.yml
services:
  api:
    deploy:
      resources:
        limits:
          memory: 2G
```

### 2.5 磁盘空间不足

**症状**: 写入失败，日志无法记录

**排查步骤**:

```bash
# 1. 检查磁盘使用
df -h

# 2. 查找大文件
du -sh /* 2>/dev/null | sort -hr | head -10

# 3. Docker 磁盘使用
docker system df

# 4. 查找日志文件
find /var/lib/docker/containers -name "*.log" -exec ls -lh {} \;
```

**解决方案**:

```bash
# 清理 Docker
docker system prune -af --volumes

# 清理日志文件
truncate -s 0 /var/lib/docker/containers/*/*.log

# 清理旧备份
find /backups -name "*.sql.gz" -mtime +7 -delete

# 设置日志轮转
# 在 /etc/docker/daemon.json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

## 3. 数据库故障排查

### 3.1 数据库连接失败

**错误信息**:
```
could not connect to server: Connection refused
```

**排查步骤**:

```bash
# 1. 检查数据库容器状态
docker-compose ps db

# 2. 检查数据库日志
docker-compose logs db

# 3. 测试数据库连接
docker-compose exec db pg_isready -U aigc

# 4. 检查连接配置
docker-compose exec api env | grep DATABASE
```

**解决方案**:

```bash
# 重启数据库
docker-compose restart db

# 重新初始化数据库
docker-compose exec db psql -U postgres -c "CREATE DATABASE aigc_prod;"
docker-compose exec api alembic upgrade head
```

### 3.2 数据库性能问题

**症状**: 查询响应缓慢

**排查命令**:

```sql
-- 查看当前活动查询
SELECT pid, usename, query, state, now() - query_start AS duration
FROM pg_stat_activity
WHERE state != 'idle'
ORDER BY duration DESC;

-- 查看锁等待
SELECT blocked.pid, blocked.query, blocking.pid AS blocking_id, blocking.query AS blocking_query
FROM pg_stat_activity blocked
JOIN pg_stat_activity blocking ON blocking.pid = ANY(pg_blocking_pids(blocked.pid));

-- 查看表大小
SELECT relname, pg_size_pretty(pg_total_relation_size(relid))
FROM pg_stat_user_tables
ORDER BY pg_total_relation_size(relid) DESC;

-- 查看索引使用
SELECT indexrelname, idx_scan, idx_tup_read, idx_tup_fetch
FROM pg_stat_user_indexes
ORDER BY idx_scan DESC;
```

**慢查询分析**:

```sql
-- 开启慢查询日志
ALTER DATABASE aigc_prod SET log_min_duration_statement = 1000;

-- 查看慢查询
SELECT start, duration, query
FROM pg_log
WHERE duration > 1000
ORDER BY start DESC;
```

### 3.3 数据库备份失败

**症状**: 备份脚本执行报错

**排查步骤**:

```bash
# 1. 测试备份命令
docker-compose exec -T db pg_dump -U aigc aigc_prod | gzip > /tmp/test_backup.sql.gz

# 2. 检查磁盘空间
df -h /backups

# 3. 检查权限
ls -la /backups

# 4. 手动执行备份
cd /app && ./scripts/backup.sh
```

## 4. Redis 故障排查

### 4.1 Redis 连接失败

**错误信息**:
```
redis.exceptions.ConnectionError: Error -2 connecting to redis:6379
```

**排查步骤**:

```bash
# 1. 检查 Redis 状态
docker-compose ps redis

# 2. 测试连接
docker-compose exec redis redis-cli ping

# 3. 查看 Redis 日志
docker-compose logs redis

# 4. 检查端口
netstat -tlnp | grep 6379
```

**解决方案**:

```bash
# 重启 Redis
docker-compose restart redis

# 清除所有数据 (谨慎使用)
docker-compose exec redis redis-cli FLUSHALL
```

### 4.2 Redis 内存问题

**症状**: Redis 响应变慢或拒绝写入

**排查命令**:

```bash
# 查看内存使用
docker-compose exec redis redis-cli info memory

# 查看 key 数量
docker-compose exec redis redis-cli DBSIZE

# 查看大 key
docker-compose exec redis redis-cli --bigkeys

# 查看内存碎片
docker-compose exec redis redis-cli info memory | grep mem_fragmentation
```

**解决方案**:

```bash
# 配置内存限制 (redis.conf)
maxmemory 256mb
maxmemory-policy allkeys-lru

# 重启应用清理连接
docker-compose restart api

# 检查是否有大 value
docker-compose exec redis redis-cli --scan | head -100 | xargs -I{} redis-cli DEBUG SLEEP 0.1 OBJECT ENCODING {}
```

### 4.3 Redis 持久化问题

**排查命令**:

```bash
# 查看持久化状态
docker-compose exec redis redis-cli info persistence

# 检查 RDB 文件
ls -la /var/lib/docker/volumes/*/data/dump.rdb

# 测试持久化
docker-compose exec redis redis-cli BGSAVE
```

## 5. 监控故障排查

### 5.1 Prometheus 无法采集指标

**症状**: Grafana 显示 "No data"

**排查步骤**:

```bash
# 1. 检查 Prometheus 状态
docker-compose -f docker-compose.ops.yml ps prometheus

# 2. 检查 Prometheus 日志
docker-compose -f docker-compose.ops.yml logs prometheus

# 3. 测试指标端点
curl http://localhost:8000/metrics

# 4. 检查 Prometheus targets
curl http://localhost:9090/api/v1/targets
```

**常见问题**:

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 403 Forbidden | 认证问题 | 检查 Prometheus 配置 |
| Connection refused | 目标服务未启动 | 启动目标服务 |
| 指标为空 | 应用未暴露 /metrics | 检查应用代码 |

### 5.2 Grafana 面板无数据

**排查步骤**:

```bash
# 1. 检查数据源配置
curl http://localhost:3000/api/datasources

# 2. 测试 Prometheus 查询
curl -G "http://localhost:9090/api/v1/query" \
  --data-urlencode 'query=http_requests_total'

# 3. 检查 Grafana 日志
docker-compose -f docker-compose.ops.yml logs grafana
```

**解决方案**:

```bash
# 重置数据源
curl -X DELETE http://localhost:3000/api/datasources/1
# 重新添加数据源

# 清除缓存
rm -rf /var/lib/grafana/cache
docker-compose -f docker-compose.ops.yml restart grafana
```

### 5.3 告警未触发

**排查步骤**:

```bash
# 1. 检查 Alertmanager 状态
curl http://localhost:9093/api/v1/status

# 2. 查看活跃告警
curl http://localhost:9093/api/v1/alerts

# 3. 检查 Prometheus 告警规则
curl http://localhost:9090/api/v1/rules | jq '.data.groups[].rules[] | select(.type=="alerting")'

# 4. 测试告警规则
curl -G "http://localhost:9090/api/v1/query" \
  --data-urlencode 'query=up{job="api"}'
```

## 8. 性能问题诊断

### 8.1 性能分析流程

```
1. 确认问题范围
   ├── 单个接口慢还是全部接口慢
   ├── 偶发问题还是持续问题
   └── 有无特定触发条件

2. 分层排查
   ├── 网络层: 延迟、带宽
   ├── 应用层: 代码逻辑、算法
   ├── 数据库层: 查询优化、索引
   └── 缓存层: 命中率、过期策略

3. 使用工具分析
   ├── Profiling: 代码级分析
   ├── APM: 分布式追踪
   └── 日志分析: 访问模式
```

### 8.2 慢查询诊断

**使用 EXPLAIN 分析**:

```sql
EXPLAIN ANALYZE
SELECT u.*, c.name as course_name
FROM users u
JOIN enrollments e ON u.id = e.user_id
JOIN courses c ON e.course_id = c.id
WHERE u.created_at > '2025-01-01';
```

**常见问题**:

| 问题 | 解决方案 |
|------|----------|
| 全表扫描 | 添加索引 |
| 索引未使用 | 重写查询或重建索引 |
| 大结果集 | 分页查询 |
| 锁等待 | 优化事务或减少锁范围 |

### 8.3 内存泄漏检测

**排查命令**:

```bash
# 监控内存增长
docker stats api --no-stream

# 查看进程内存
docker-compose exec api ps aux --sort=-rss

# 检查内存分配
docker-compose exec api python -c "
import tracemalloc
tracemalloc.start()
# 运行代码
snapshot = tracemalloc.take_snapshot()
top_stats = snapshot.statistics('lineno')
for stat in top_stats[:10]:
    print(stat)
"
```

## 9. 紧急故障处理

### 9.1 服务完全不可用

**紧急响应流程**:

```
1. 立即行动 (0-5分钟)
   ├── 确认故障范围
   ├── 检查服务状态
   └── 准备回滚方案

2. 临时修复 (5-15分钟)
   ├── 重启服务
   ├── 启用备用实例
   └── 限制流量

3. 根因分析 (15-60分钟)
   ├── 收集日志
   ├── 分析根因
   └── 制定修复方案

4. 后续跟进
   ├── 编写故障报告
   ├── 实施长期修复
   └── 更新监控告警
```

### 9.2 数据损坏处理

**警告**: 以下操作可能导致数据丢失，务必谨慎！

```bash
# 1. 立即停止写入
docker-compose stop api

# 2. 备份当前数据
docker-compose exec -T db pg_dump -U aigc aigc_prod > /backups/emergency_backup_$(date +%Y%m%d_%H%M%S).sql

# 3. 检查损坏范围
docker-compose exec db psql -U aigc -c "SELECT * FROM corrupted_table LIMIT 10;"

# 4. 从备份恢复
gunzip < /backups/aigc_prod_20250509_020000.sql.gz | docker-compose exec -T db psql -U aigc aigc_prod

# 5. 验证数据完整性
docker-compose exec db psql -U aigc -c "SELECT count(*) FROM users;"
```

## 10. 预防措施

### 10.1 日常巡检

```bash
#!/bin/bash
# daily-check.sh - 每日巡检脚本

echo "=== Aigc For Study 每日巡检 $(date) ==="

echo -e "\n[1] 服务状态检查"
docker-compose ps

echo -e "\n[2] 资源使用检查"
docker stats --no-stream --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}\t{{.MemPerc}}"

echo -e "\n[3] 磁盘空间检查"
df -h / /var/lib/docker

echo -e "\n[4] 活跃告警检查"
curl -s http://localhost:9093/api/v1/alerts | jq '.data[] | select(.state=="firing") | .labels.alertname'

echo -e "\n[5] 错误日志检查 (最近 1 小时)"
docker-compose logs --since=1h api | grep -i error | tail -20

echo -e "\n[6] 数据库连接检查"
docker-compose exec db pg_isready -U aigc

echo -e "\n[7] Redis 连接检查"
docker-compose exec redis redis-cli ping

echo -e "\n=== 巡检完成 ==="
```
