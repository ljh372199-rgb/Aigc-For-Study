## 1. 指标概述

### 1.1 指标体系架构

```
┌─────────────┐
│  应用层     │  http_requests_total, http_request_duration_seconds
├─────────────┤
│  服务层     │  AI API 调用次数、延迟、成本
├─────────────┤
│  中间件层   │  PostgreSQL, Redis 连接池、查询性能
├─────────────┤
│  系统层     │  CPU, Memory, Disk, Network
├─────────────┤
│  容器层     │  Container CPU, Memory, Network, Block IO
└─────────────┘
```

### 1.2 指标命名规范

指标遵循以下命名规范：

| 前缀 | 说明 | 示例 |
|------|------|------|
| `http_` | HTTP 相关指标 | `http_requests_total` |
| `db_` | 数据库相关指标 | `db_connections_active` |
| `redis_` | Redis 相关指标 | `redis_memory_used_bytes` |
| `ai_` | AI API 相关指标 | `ai_api_requests_total` |
| `node_` | 系统指标 (Node Exporter) | `node_cpu_seconds_total` |
| `container_` | 容器指标 (cAdvisor) | `container_memory_usage_bytes` |

### 1.3 指标类型

| 类型 | 说明 | 示例 |
|------|------|------|
| Counter | 只能递增的计数器 | 请求总数、错误总数 |
| Gauge | 可增可减的数值 | CPU 使用率、内存使用量 |
| Histogram | 统计分布 | 请求延迟、响应大小 |
| Summary | 百分位统计 | P50/P95/P99 延迟 |

## 3. HTTP 指标

### 3.1 请求总量

**指标名称**: `http_requests_total`

**类型**: Counter

**标签**:
| 标签 | 说明 | 示例 |
|------|------|------|
| `job` | 任务名 | `api` |
| `instance` | 实例地址 | `api:8000` |
| `method` | HTTP 方法 | `GET`, `POST`, `PUT`, `DELETE` |
| `endpoint` | API 端点 | `/api/v1/courses` |
| `status` | HTTP 状态码 | `200`, `404`, `500` |

**查询示例**:

```promql
# 总请求数
sum(http_requests_total{job="api"})

# 按状态码统计
sum by (status) (http_requests_total{job="api"})

# 按端点统计 TOP 10
topk(10, sum by (endpoint) (rate(http_requests_total{job="api"}[5m])))

# QPS (每秒请求数)
sum(rate(http_requests_total{job="api"}[5m]))
```

### 3.2 请求延迟

**指标名称**: `http_request_duration_seconds`

**类型**: Histogram

**标签**: `job`, `instance`, `method`, `endpoint`, `status`

**Buckets**: `[0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1, 2.5, 5, 10]`

**查询示例**:

```promql
# 平均延迟
sum(rate(http_request_duration_seconds_sum{job="api"}[5m])) /
sum(rate(http_request_duration_seconds_count{job="api"}[5m]))

# P50 延迟
histogram_quantile(0.50, sum(rate(http_request_duration_seconds_bucket{job="api"}[5m])) by (le))

# P95 延迟
histogram_quantile(0.95, sum(rate(http_request_duration_seconds_bucket{job="api"}[5m])) by (le))

# P99 延迟
histogram_quantile(0.99, sum(rate(http_request_duration_seconds_bucket{job="api"}[5m])) by (le))

# 特定端点延迟
histogram_quantile(0.95,
  sum(rate(http_request_duration_seconds_bucket{job="api", endpoint="/api/v1/courses"}[5m])) by (le)
)
```

### 3.3 错误率

**指标名称**: `http_requests_total`

**计算公式**:

```
错误率 = sum(rate(http_requests_total{status=~"4..|5.."}[5m])) /
         sum(rate(http_requests_total[5m]))
```

**查询示例**:

```promql
# 5xx 错误率
sum(rate(http_requests_total{job="api", status=~"5.."}[5m])) /
sum(rate(http_requests_total{job="api"}[5m]))

# 4xx 错误率
sum(rate(http_requests_total{job="api", status=~"4.."}[5m])) /
sum(rate(http_requests_total{job="api"}[5m]))

# 错误数 (5分钟内)
sum(increase(http_requests_total{job="api", status=~"5.."}[5m]))
```

### 3.4 HTTP 状态码分布

| 状态码范围 | 含义 | 告警阈值 |
|------------|------|----------|
| 2xx | 成功 | - |
| 4xx | 客户端错误 | > 10% |
| 5xx | 服务器错误 | > 1% |

**查询示例**:

```promql
# 各状态码请求数分布
sum by (status) (increase(http_requests_total{job="api"}[1h]))

# 状态码占比
sum by (status) (rate(http_requests_total{job="api"}[5m])) /
sum(rate(http_requests_total{job="api"}[5m]))
```

## 4. 数据库指标

### 4.1 连接指标

**指标名称**: `db_connections`

| 指标 | 说明 | 类型 |
|------|------|------|
| `db_connections_total` | 连接总数 | Counter |
| `db_connections_active` | 活跃连接数 | Gauge |
| `db_connections_idle` | 空闲连接数 | Gauge |
| `db_connections_max` | 最大连接数 | Gauge |

**当前配置**:
| 参数 | 开发环境 | 生产环境 |
|------|----------|----------|
| 连接池大小 | 10 | 20 |
| 最大溢出 | 20 | 40 |
| 连接超时 | 30s | 30s |

**查询示例**:

```promql
# 活跃连接数
db_connections_active{pool="default"}

# 连接池使用率
db_connections_active / db_connections_max * 100

# 连接等待时间
db_connection_wait_seconds_total
```

### 4.2 查询性能指标

**指标名称**: `db_query_duration_seconds`

**类型**: Histogram

**标签**: `pool`, `operation` (SELECT, INSERT, UPDATE, DELETE)

**Buckets**: `[0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1, 5]`

**查询示例**:

```promql
# 平均查询时间
sum(rate(db_query_duration_seconds_sum[5m])) /
sum(rate(db_query_duration_seconds_count[5m]))

# 慢查询 (> 100ms)
histogram_quantile(0.95,
  sum(rate(db_query_duration_seconds_bucket[5m])) by (le)
)

# 按操作类型统计
sum by (operation) (rate(db_query_duration_seconds_count[5m]))
```

### 4.3 事务指标

**指标名称**: `db_transaction_duration_seconds`

**查询示例**:

```promql
# 事务平均执行时间
sum(rate(db_transaction_duration_seconds_sum[5m])) /
sum(rate(db_transaction_duration_seconds_count[5m]))

# 事务数量
sum(rate(db_transaction_duration_seconds_count[5m]))
```

### 4.4 健康指标

```promql
# 数据库可访问性
up{job="postgresql"}

# 连接池健康
db_connections_active / db_connections_max < 0.9
```

## 5. Redis 指标

### 5.1 内存指标

**指标名称**: `redis_memory_used_bytes`

**类型**: Gauge

**查询示例**:

```promql
# 内存使用量
redis_memory_used_bytes{instance="redis:6379"}

# 内存使用率
redis_memory_used_bytes / redis_memory_max_bytes * 100

# 内存碎片率
redis_mem_fragmentation_ratio
```

**告警阈值**:
| 指标 | 警告 | 严重 |
|------|------|------|
| 内存使用率 | > 70% | > 85% |
| 内存碎片率 | > 1.5 | > 2.0 |

### 5.2 命中率指标

**指标名称**: `redis_keyspace_hits_total`, `redis_keyspace_misses_total`

**查询示例**:

```promql
# 命中率
redis_keyspace_hits_total / (redis_keyspace_hits_total + redis_keyspace_misses_total) * 100

# 命中/未命中数
sum(redis_keyspace_hits_total)
sum(redis_keyspace_misses_total)
```

**告警阈值**:
| 指标 | 警告 | 严重 |
|------|------|------|
| 命中率 | < 80% | < 60% |

### 5.3 连接指标

**指标名称**: `redis_connected_clients`

**查询示例**:

```promql
# 客户端连接数
redis_connected_clients

# 已阻止客户端数
redis_blocked_clients

# 最大连接数
redis_maxclients
```

### 5.4 持久化指标

**指标名称**: `redis_rdb_changes_since_last_save`, `redis_aof_current_size`

**查询示例**:

```promql
# 最后一次 RDB 保存后的变更数
redis_rdb_changes_since_last_save

# AOF 文件大小
redis_aof_current_size

# AOF 重写状态
redis_aof_rewrite_in_progress
```

### 5.5 命令统计

**指标名称**: `redis_commands_total`

**标签**: `cmd` (get, set, mget, etc.)

**查询示例**:

```promql
# 命令执行速率
sum by (cmd) (rate(redis_commands_total[5m]))

# 最常用命令 TOP 10
topk(10, sum by (cmd) (rate(redis_commands_total[5m])))
```

### 5.6 驱逐指标

**指标名称**: `redis_evicted_keys_total`

**查询示例**:

```promql
# 驱逐速率
rate(redis_evicted_keys_total[5m])

# 驱逐总数
redis_evicted_keys_total
```

## 6. AI API 指标

### 6.1 请求指标

**指标名称**: `ai_api_requests_total`

**类型**: Counter

**标签**:
| 标签 | 说明 | 示例 |
|------|------|------|
| `provider` | AI 提供商 | `openai`, `claude`, `deepseek` |
| `model` | 模型名称 | `gpt-4-turbo`, `claude-3-opus` |
| `operation` | 操作类型 | `chat`, `completion` |
| `status` | 调用状态 | `success`, `error`, `timeout` |

**查询示例**:

```promql
# 总请求数
sum(ai_api_requests_total)

# 按提供商统计
sum by (provider) (ai_api_requests_total)

# 成功率
sum(ai_api_requests_total{status="success"}) /
sum(ai_api_requests_total) * 100

# 错误率
sum by (provider, status) (rate(ai_api_requests_total{status!="success"}[5m]))
```

### 6.2 延迟指标

**指标名称**: `ai_api_request_duration_seconds`

**类型**: Histogram

**标签**: `provider`, `model`, `operation`

**Buckets**: `[0.5, 1, 2, 5, 10, 20, 30, 60]`

**查询示例**:

```promql
# 平均延迟
sum(rate(ai_api_request_duration_seconds_sum[5m])) /
sum(rate(ai_api_request_duration_seconds_count[5m]))

# P95 延迟
histogram_quantile(0.95,
  sum(rate(ai_api_request_duration_seconds_bucket[5m])) by (le, provider)
)

# 各提供商延迟对比
sum by (provider) (
  sum(rate(ai_api_request_duration_seconds_sum[5m])) by (provider) /
  sum(rate(ai_api_request_duration_seconds_count[5m])) by (provider)
)
```

### 6.3 Token 使用指标

**指标名称**: `ai_api_tokens_total`

**类型**: Counter

**标签**: `provider`, `model`, `type` (input, output)

**查询示例**:

```promql
# 总 Token 消耗
sum(ai_api_tokens_total)

# 按提供商统计
sum by (provider) (ai_api_tokens_total)

# 输入/输出 Token 比例
sum by (type) (rate(ai_api_tokens_total[1h]))

# Token 消耗趋势
sum by (provider) (rate(ai_api_tokens_total[5m]))
```

### 6.4 成本指标

**指标名称**: `ai_api_cost_total`

**类型**: Counter

**标签**: `provider`, `model`

**单位**: 美元

**查询示例**:

```promql
# 总成本
sum(ai_api_cost_total)

# 各提供商成本占比
sum by (provider) (ai_api_cost_total) / sum(ai_api_cost_total) * 100

# 每小时成本趋势
sum by (provider) (increase(ai_api_cost_total[1h]))
```

### 6.5 配额指标

**指标名称**: `ai_api_quota_remaining`

**类型**: Gauge

**标签**: `provider`

**查询示例**:

```promql
# 剩余配额
ai_api_quota_remaining{provider="openai"}

# 配额使用率
(ai_api_quota_limit - ai_api_quota_remaining) / ai_api_quota_limit * 100
```

## 7. 系统指标

### 7.1 CPU 指标

**指标来源**: Node Exporter

**指标名称**: `node_cpu_seconds_total`

**类型**: Counter

**标签**: `mode` (user, system, idle, iowait, etc.)

**查询示例**:

```promql
# CPU 使用率
100 - (avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# 各模式 CPU 使用
sum by (mode) (rate(node_cpu_seconds_total[5m])) * 100

# 按核心统计
avg by (instance, cpu) (rate(node_cpu_seconds_total{mode!="idle"}[5m])) * 100
```

**告警阈值**:
| 指标 | 警告 | 严重 |
|------|------|------|
| CPU 使用率 | > 70% | > 85% |
| CPU iowait | > 20% | > 40% |

### 7.2 内存指标

**指标名称**: `node_memory_*`

**查询示例**:

```promql
# 内存使用率
(node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) /
node_memory_MemTotal_bytes * 100

# 已用内存
node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes

# 可用内存
node_memory_MemAvailable_bytes

# 交换区使用
node_memory_SwapTotal_bytes - node_memory_SwapFree_bytes
```

**告警阈值**:
| 指标 | 警告 | 严重 |
|------|------|------|
| 内存使用率 | > 75% | > 90% |
| 交换区使用 | > 50% | > 80% |

### 7.3 磁盘指标

**指标名称**: `node_disk_*`

**查询示例**:

```promql
# 磁盘使用率
100 - (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"} * 100)

# 磁盘 IOPS
rate(node_disk_reads_completed_total[5m])
rate(node_disk_writes_completed_total[5m])

# 磁盘吞吐量
rate(node_disk_read_bytes_total[5m])
rate(node_disk_writes_bytes_total[5m])

# IO wait
rate(node_cpu_seconds_total{mode="iowait"}[5m]) * 100
```

**告警阈值**:
| 指标 | 警告 | 严重 |
|------|------|------|
| 磁盘使用率 | > 75% | > 85% |
| inode 使用率 | > 80% | > 90% |

### 7.4 网络指标

**指标名称**: `node_network_*`

**查询示例**:

```promql

## 9. 告警阈值配置

### 9.1 告警规则文件

告警规则位于 `ops/prometheus/alert_rules/` 目录：

```
ops/prometheus/alert_rules/
├── README.md           # 规则说明
├── aigc_alerts.yml     # 业务告警
├── api_alerts.yml      # API 告警

### 9.2 API 告警规则

```yaml
groups:
  - name: api_alerts
    interval: 30s
    rules:
      # 服务不可用
      - alert: APIServiceDown
        expr: up{job="api"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "API 服务不可用"
          description: "实例 {{ $labels.instance }} 已停止运行超过 1 分钟"

      # 高错误率
      - alert: HighAPIErrorRate
        expr: |
          sum(rate(http_requests_total{job="api", status=~"5.."}[5m])) /
          sum(rate(http_requests_total{job="api"}[5m])) > 0.01
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "API 5xx 错误率超过 1%"
          description: "当前错误率: {{ $value | humanizePercentage }}"

      # 响应时间过长
      - alert: HighAPIResponseTime
        expr: |
          histogram_quantile(0.95,
            sum(rate(http_request_duration_seconds_bucket{job="api"}[5m])) by (le)
          ) > 1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "API 响应时间过长"
          description: "P95 响应时间超过 1 秒"
```

### 9.3 系统告警规则

```yaml
groups:
  - name: system_alerts
    interval: 30s
    rules:
      # CPU 使用率过高
      - alert: HighCPUUsage
        expr: |
          100 - (avg by (instance)(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "CPU 使用率超过 80%"
          description: "服务器 {{ $labels.instance }} CPU 使用率: {{ $value | humanize }}"

      # 内存使用率过高
      - alert: HighMemoryUsage
        expr: |
          (node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) /
          node_memory_MemTotal_bytes * 100 > 85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "内存使用率超过 85%"
          description: "服务器 {{ $labels.instance }} 内存使用率: {{ $value | humanize }}"

      # 磁盘空间不足
      - alert: DiskSpaceLow
        expr: |
          (node_filesystem_avail_bytes{mountpoint="/"} /
           node_filesystem_size_bytes{mountpoint="/"}) * 100 < 15
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "磁盘空间不足"
          description: "服务器 {{ $labels.instance }} 根分区可用空间低于 15%"
```

## 10. PromQL 查询示例

### 10.1 基础查询

```promql
# 查询所有指标
{__name__=~"http_.*"}

# 过滤特定标签
http_requests_total{job="api", status="200"}

# 使用正则匹配
http_requests_total{endpoint=~"/api/v1/.*"}
```

### 10.2 聚合查询

```promql
# 按标签聚合
sum by (status) (http_requests_total)

# 不按标签聚合
sum without (instance) (http_requests_total)

# 计算平均值
avg(http_request_duration_seconds)

# 计算最大值
max by (endpoint) (http_request_duration_seconds)
```

### 10.3 时间函数

```promql
# 过去 5 分钟的增长率
rate(http_requests_total[5m])

# 过去 1 小时的增量
increase(http_requests_total[1h])

# 指定时间点的值
http_requests_total at 1704067200
```

### 10.4 常用业务查询

```promql
# 当前在线用户数
sum(active_users)

# 今日 API 调用量
sum(increase(http_requests_total{job="api"}[24h]))

# API 可用性 (SLA)
sum(rate(http_requests_total{status=~"2.."}[1h])) /
sum(rate(http_requests_total[1h])) * 100

```

## 11. 最佳实践

### 11.1 指标采集优化

1. **合理设置 scrape interval**
   - 高频指标: 15s
   - 低频指标: 30s 或 1m

2. **控制指标 cardinality**
   - 避免高基数标签 (user_id, request_id)
   - 使用 `label_values()` 预聚合

3. **历史数据保留**
   - Raw data: 15 天
   - 5 分钟采样: 60 天
   - 1 小时采样: 1 年

### 11.2 告警优化

1. **避免告警风暴**
   - 使用 `group_by` 聚合
   - 设置 `group_wait` 和 `group_interval`

2. **合理设置持续时间**
   - 瞬时问题: `for: 0m`
   - 持续问题: `for: 5m`

3. **分级告警**
   - Critical: 立即响应
   - Warning: 关注处理
   - Info: 记录分析

### 11.3 查询优化

1. **使用 `rate()` 而非 `increase()`**
   ```promql
   # 推荐
   rate(http_requests_total[5m])
   
   # 不推荐
   increase(http_requests_total[5m]) / 300
   ```

2. **避免全量扫描**
   ```promql
   # 推荐: 添加标签过滤
   http_requests_total{job="api"}
   
   # 不推荐: 无过滤
   http_requests_total
   ```

3. **使用预聚合**
   ```promql
   # 提前聚合减少计算量
   sum(rate(http_requests_total[5m])) by (status)
   ```
