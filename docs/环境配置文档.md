## 1. 环境概述

### 1.1 环境分层

| 环境名称 | 用途 | 域名示例 | 数据库 | 资源配置 |
|----------|------|----------|--------|----------|
| 开发环境 (Development) | 本地开发调试 | localhost | 本地PostgreSQL | 本机资源 |
| 测试环境 (Staging) | 功能测试、集成测试 | staging.aigcstudy.com | 测试数据库 | 4核4G |
| 生产环境 (Production) | 正式运行 | aigcstudy.com | 生产数据库 | 4核4G |

### 1.2 环境配置原则

1. **配置分离**: 代码与配置分离，敏感信息不进入代码仓库
2. **环境隔离**: 各环境数据库、缓存完全隔离
3. **最小权限**: 各环境使用独立的数据库账号和API密钥
4. **可追溯性**: 所有配置变更记录在案

## 2. 开发环境配置

### 2.1 系统要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| Docker | 24.0+ | 容器运行环境 |
| Docker Compose | 2.20+ | 容器编排 |
| Git | 2.40+ | 版本控制 |
| Node.js | 18.x LTS | 前端开发 |
| Python | 3.11+ | 后端开发 |

### 2.2 本地环境搭建

**步骤1: 克隆代码仓库**

```bash
git clone https://github.com/xxx/aigc-for-study.git
cd aigc-for-study
```

**步骤2: 创建环境变量文件**

```bash
# 后端环境变量
cp backend/.env.example backend/.env

# 前端环境变量
cp frontend/.env.example frontend/.env
```

**步骤3: 启动开发服务**

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

**步骤4: 初始化数据库**

```bash
# 进入后端容器
docker-compose exec api bash

# 执行数据库迁移
alembic upgrade head

# 初始化种子数据（可选）
python scripts/init_data.py
```

## 5. 环境变量管理

### 5.1 变量分类

| 类别 | 存储位置 | 示例 |
|------|----------|------|
| 公开配置 | 代码仓库 (.env.example) | APP_NAME, LOG_LEVEL |
| 敏感配置 | 密钥管理系统 | SECRET_KEY, API_KEY |
| 环境特定 | CI/CD环境变量 | DATABASE_URL, REDIS_URL |

### 5.2 密钥管理方案

**MVP阶段（当前）**

使用 `.env` 文件 + `.gitignore` 排除：

```bash
# .gitignore
.env
.env.local
.env.*.local
*.pem
*.key
```

**后续扩展（甲方接管后）**

建议使用专业密钥管理工具：

| 方案 | 适用场景 | 成本 |
|------|----------|------|
| HashiCorp Vault | 自建密钥管理 | 中 |
| AWS Secrets Manager | AWS云环境 | 按量计费 |
| 阿里云KMS | 阿里云环境 | 按量计费 |
| GitHub Secrets | CI/CD场景 | 免费 |

### 5.3 环境变量注入方式

**方式1: Docker Compose环境变量**

```yaml
services:
  api:
    environment:
      - APP_ENV=production
    env_file:
      - .env.production
```

**方式2: 系统环境变量**

```bash
# /etc/environment
export DATABASE_URL="postgresql://..."
export SECRET_KEY="xxx"
```

**方式3: CI/CD注入**

```yaml
# GitHub Actions
- name: Deploy
  env:
    DATABASE_URL: ${{ secrets.DATABASE_URL }}
    SECRET_KEY: ${{ secrets.SECRET_KEY }}
```

  - name: aigc-alerts
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.1
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "High error rate detected"

      - alert: HighMemoryUsage

        for: 1m
        labels:

## 9. 备份配置

### 9.1 数据库备份脚本

**backup.sh**

```bash
#!/bin/bash

# 配置
DB_NAME="aigc_prod"
DB_USER="aigc"
BACKUP_DIR="/backups"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/${DB_NAME}_${DATE}.sql.gz"

# 创建备份目录
mkdir -p ${BACKUP_DIR}

# 执行备份
docker-compose exec -T db pg_dump -U ${DB_USER} ${DB_NAME} | gzip > ${BACKUP_FILE}

# 删除7天前的备份
find ${BACKUP_DIR} -name "*.sql.gz" -mtime +7 -delete

# 记录日志
echo "Backup completed: ${BACKUP_FILE}" >> ${BACKUP_DIR}/backup.log
```

### 9.2 定时任务配置

```bash
# crontab -e

# 每日凌晨2点全量备份
0 2 * * * /app/scripts/backup.sh

# 每小时增量备份
0 * * * * /app/scripts/incremental_backup.sh
```
