# Aigc For Study 统一运维平台规范

**版本:** v2.0
**制定日期:** 2026-05-09
**项目背景:** 基于 Aigc For Study AI学习平台构建自主开发的统一运维平台，整合监控、告警、日志功能，提供苹果风格的优秀用户体验

---

## 1. 为什么需要统一运维平台

### 1.1 现有方案的问题

当前基于开源组件（Grafana + Prometheus + Loki + Alertmanager）的方案存在以下问题：

- **体验割裂**: 用户需要在多个界面之间跳转，认知负担重
- **界面不统一**: 每个开源工具都有自己风格，学习成本高
- **功能分散**: 监控、告警、日志分布在不同系统中
- **定制困难**: 开源工具的主题定制能力有限

### 1.2 统一平台的优势

- **单一入口**: 所有运维功能在一个界面中完成
- **一致体验**: 统一的交互设计和视觉风格
- **深度集成**: 监控、告警、日志无缝关联
- **完全可控**: UI 和交互完全自主可控

## 2. 设计理念：苹果风格

### 2.1 视觉语言

**核心原则**：
- **简洁克制**: 极简设计，减少视觉噪音
- **内容优先**: 内容是主角，UI 是配角
- **优雅动效**: 微妙的过渡和反馈
- **一致系统**: 统一的设计语言和组件库

**色彩系统**：
```
主色系 (Dark Mode):
- 背景色: #000000 (纯黑) / #1d1d1f (深灰)
- 表面色: #2d2d2f (卡片背景)
- 边框色: #3d3d3f (微妙分隔)
- 主文字: #f5f5f7 (高对比白)
- 次文字: #86868b (柔和灰)

强调色:
- 蓝色: #0a84ff (主要操作)
- 绿色: #30d158 (健康/成功)
- 黄色: #ffd60a (警告)
- 红色: #ff453a (错误/严重)
- 紫色: #bf5af2 (特殊标记)

功能色:
- 监控指标: #30d158 (绿色系)
- 告警提示: #ff453a (红色系)
- 日志内容: #0a84ff (蓝色系)
```

**字体系统**：
```
主字体: SF Pro Display / -apple-system / BlinkMacSystemFont
- 标题 H1: 28px, 600 weight
- 标题 H2: 22px, 600 weight
- 标题 H3: 17px, 600 weight
- 正文: 14px, 400 weight
- 标签: 12px, 500 weight
- 数值: SF Mono / Menlo (等宽)
```

**间距系统**：
```
4px 基础单位
- xs: 4px   (紧凑元素)
- sm: 8px   (组件内间距)
- md: 12px  (元素间距)
- lg: 16px  (区块间距)
- xl: 24px  (区域间距)
- xxl: 32px (页面边距)
```

**圆角系统**：
```
- 小圆角: 6px  (按钮、输入框)
- 中圆角: 10px (卡片、面板)
- 大圆角: 14px (模态框、容器)
```

### 2.2 组件设计

**按钮**：
- 高度: 32px (小) / 40px (中) / 48px (大)
- 内边距: 12px 水平, 8px 垂直
- 过渡: 150ms ease-out
- Hover: 亮度 +5%
- Active: 亮度 -5%

**卡片**：
- 背景: #1d1d1f
- 圆角: 10px
- 内边距: 16px
- 边框: 1px solid #3d3d3f
- 悬停: 边框变为 #0a84ff

**输入框**：
- 高度: 36px
- 背景: #2d2d2f
- 边框: 1px solid #3d3d3f
- 聚焦: 边框变为 #0a84ff
- 圆角: 6px

**图表**：
- 背景透明，坐标轴使用次文字色
- 数据点使用主题强调色
- 网格线使用 #3d3d3f
- 动画: 300ms ease-out

### 2.3 交互设计

**导航**：
- 侧边栏固定宽度 220px
- 图标 + 文字组合
- 悬停效果: 背景微亮
- 当前状态: 左侧蓝色指示条

**页面过渡**：
- 使用 Framer Motion
- 淡入淡出 + 轻微位移
- 持续时间: 200ms
- 缓动: ease-out

**加载状态**：
- 使用毛玻璃效果
- Skeleton 骨架屏
- 优雅的脉冲动画

**空状态**：
- 简洁插图 + 说明文字
- 引导性操作按钮

## 5. 技术实现

### 5.1 技术栈

**前端**：
- React 18 + TypeScript
- Tailwind CSS (自定义配置)
- Framer Motion (动画)
- Recharts (图表)
- Zustand (状态管理)

**后端**：
- FastAPI (Python)
- PostgreSQL (元数据存储)
- Redis (缓存和队列)

**数据层**：
- Prometheus (指标存储和查询)
- Loki (日志存储和查询)
- Alertmanager (告警聚合)

### 5.2 API 设计

```typescript
// 统一 API 响应格式
interface ApiResponse<T> {
  code: number;        // 0: 成功, 其他: 错误码
  message: string;     // 消息
  data: T;            // 数据
  timestamp: number;  // 时间戳
}

// 指标查询
GET /api/v1/metrics/query
Query: {
  metric: string;      // 指标名
  start: number;       // 开始时间戳
  end: number;         // 结束时间戳
  step: string;        // 步长 (如 15s, 1m)
  filters?: Record<string, string>;  // 标签过滤
}

// 告警列表
GET /api/v1/alerts
Query: {
  status?: 'firing' | 'resolved';  // 状态
  severity?: string[];            // 严重程度
  page?: number;                   // 页码
  pageSize?: number;              // 每页数量
}

// 日志查询
GET /api/v1/logs
Query: {
  query: string;       // LogQL 查询
  start: number;       // 开始时间
  end: number;         // 结束时间
  limit?: number;      // 返回条数
  direction?: 'forward' | 'backward';
}

// 服务列表
GET /api/v1/services
Response: {
  services: Service[];
}

interface Service {
  name: string;
  status: 'healthy' | 'warning' | 'critical';
  endpoints: Endpoint[];
  metrics: ServiceMetrics;
}
```
