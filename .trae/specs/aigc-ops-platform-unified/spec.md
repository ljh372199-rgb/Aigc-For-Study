# 统一运维平台构建规范

**版本:** v2.0
**制定日期:** 2026-05-09
**项目背景:** 构建统一的苹果风格运维平台，整合监控、告警、日志功能
**设计风格:** Apple Human Interface Guidelines (极简、留白、流畅)

---

## 1. 项目概述

### 1.1 核心目标

构建一个**统一的苹果风格运维平台**，摒弃割裂的传统监控体验，将监控、告警、日志三大功能整合到同一个界面中，提供流畅、直观、高效的运维管理体验。

### 1.2 设计理念

- **极简主义**: Apple 一贯的设计哲学 - 少即是多
- **信息密度适中**: 关键信息一目了然，次要信息按需展开
- **流畅动效**: 微妙的过渡动画，提升用户体验
- **一致的视觉语言**: 统一的色彩、间距、圆角规范
- **单页应用**: 所有功能在同一个界面中无缝切换

### 1.3 统一平台 vs 传统方案对比

| 维度 | 传统方案 (割裂) | 统一平台 (本方案) |
|------|----------------|-------------------|
| 界面数量 | 5+ (Prometheus, Grafana, Alertmanager, Loki, Kibana) | 1 (单一入口) |
| 认证方式 | 多套账号密码 | 统一认证 |
| 学习成本 | 高 (需熟悉各系统) | 低 (统一交互模式) |
| 数据关联 | 需手动关联 | 自动关联 |
| 品牌体验 | 分散、不一致 | 统一、专业 |

### 3.3 核心功能模块

#### 3.3.1 概览仪表盘 (Dashboard)

- **系统健康状态卡片**: 环形进度指示器，绿色/黄色/红色状态
- **实时指标**: QPS、延迟、错误率三大核心指标
- **告警摘要**: 当前活跃告警数量，按严重程度分组
- **服务状态列表**: 所有服务健康状态一览
- **最近事件时间线**: 最近 10 条关键事件

#### 3.3.2 监控中心 (Monitoring)

- **指标选择器**: 下拉选择要查看的指标
- **时间范围**: 快捷按钮 (1h, 6h, 24h, 7d) + 自定义
- **图表区域**:
  - 折线图: 指标趋势
  - 柱状图: 对比分析
  - 饼图: 占比展示
- **数据表格**: 详细指标数值，支持排序和搜索

#### 3.3.3 告警中心 (Alerts)

- **告警列表**:
  - 按状态筛选 (活跃/已解决/已静默)
  - 按级别筛选 (Critical/High/Medium/Low)
  - 按服务筛选
- **告警详情**:
  - 告警名称、级别、时间
  - 当前值 vs 阈值
  - 触发条件 (PromQL)
  - 操作按钮 (确认/静默/解决)
- **告警历史**: 告警处理记录和趋势

#### 3.3.4 日志中心 (Logs)

- **实时日志流**: WebSocket 推送的实时日志
- **日志搜索**:
  - 全文搜索
  - 字段过滤 (level, service, trace_id)
  - 时间范围选择
- **日志详情**: 展开查看完整日志 JSON
- **日志统计**: 日志量趋势图

### 4.3 API 设计

#### 4.3.1 监控指标 API

```python
# GET /api/v1/metrics
# 查询指标数据
{
  "query": "http_requests_total",
  "start": "2026-05-09T00:00:00Z",
  "end": "2026-05-09T12:00:00Z",
  "step": "1m"
}

# Response
{
  "status": "success",
  "data": {
    "metric": "http_requests_total",
    "values": [[timestamp, value], ...]
  }
}
```

#### 4.3.2 告警 API

```python
# GET /api/v1/alerts
# 获取告警列表
{
  "status": "active|resolved|silenced",
  "severity": "critical|high|medium|low",
  "service": "api|web|db"
}

# POST /api/v1/alerts/{alert_id}/silence
# 静默告警
{
  "duration": "1h",
  "reason": "计划内维护"
}
```

#### 4.3.3 日志 API

```python
# GET /api/v1/logs
# 查询日志
{
  "query": "{job=\"api\"} |= \"ERROR\"",
  "start": "2026-05-09T00:00:00Z",
  "end": "2026-05-09T12:00:00Z",
  "limit": 100
}

# WebSocket /api/v1/logs/stream
# 实时日志流
{
  "query": "{job=\"api\"}",
  "limit": 50
}
```

## 6. 交互设计

### 6.1 导航交互

- **侧边栏**: 固定在左侧，可在设置中收起
- **标签切换**: 点击标签平滑过渡，无页面刷新

# 获取告警状态
@app.get("/api/v1/alerts")
async def get_alerts():
    response = await httpx.get(
        f"{ALERTMANAGER_URL}/api/v2/alerts"
    )
    return response.json()

# 静默告警
@app.post("/api/v1/alerts/{alert_id}/silence")
async def silence_alert(alert_id: str, duration: str, reason: str):
    # 创建静默规则
    silence = {
        "matchers": [{"name": "alertname", "value": alert_id}],
        "startsAt": datetime.utcnow().isoformat(),
        "endsAt": (datetime.utcnow() + parse_duration(duration)).isoformat(),
        "comment": reason
    }
    response = await httpx.post(
        f"{ALERTMANAGER_URL}/api/v2/silences",
        json=silence
    )
    return response.json()
