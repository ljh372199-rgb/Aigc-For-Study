# Docker统一构建脚本任务清单

## 任务列表

### 任务1: 创建统一构建脚本 build-all.sh
**描述**: 创建一键构建前后端所有镜像的统一构建脚本
**优先级**: P0
**依赖**: 无
**验收标准**: 
- 支持 `--all`、`--backend`、`--frontend` 参数
- 支持 `--platforms`、`--registry`、`--tag`、`--push` 参数
- 自动检测并调用现有的后端和前端构建脚本
- 统一的日志输出格式

**子任务**:
- [ ] 创建 build-all.sh 基础框架
- [ ] 实现参数解析
- [ ] 实现后端构建调用逻辑
- [ ] 实现前端构建调用逻辑
- [ ] 实现构建结果汇总输出
- [ ] 添加错误处理

### 任务2: 创建构建服务脚本 build-services.sh
**描述**: 创建构建服务的辅助脚本
**优先级**: P1
**依赖**: 任务1
**验收标准**:
- 封装构建逻辑供其他脚本调用
- 提供构建状态查询功能

**子任务**:
- [ ] 创建 build-services.sh 基础框架
- [ ] 实现后端构建函数
- [ ] 实现前端构建函数
- [ ] 实现构建状态查询

### 任务3: 创建镜像版本管理脚本 version.sh
**描述**: 创建镜像版本管理脚本
**优先级**: P2
**依赖**: 无
**验收标准**:
- 自动从Git tag获取版本号
- 支持自定义版本号

**子任务**:
- [ ] 创建 version.sh 基础框架
- [ ] 实现Git tag版本获取
- [ ] 实现版本号格式化

## 任务依赖关系

```
任务1 (build-all.sh)
    ↓
任务2 (build-services.sh) 依赖 任务1
    ↓
任务3 (version.sh) 无依赖，可并行
```

## 验证标准

1. `build-all.sh --help` 显示正确帮助信息
2. `build-all.sh --backend --tag test` 仅构建后端镜像
3. `build-all.sh --frontend --tag test` 仅构建前端镜像
4. `build-all.sh --all --tag test` 同时构建前后端镜像
5. `build-all.sh --all --tag test --push` 构建并推送镜像
6. 所有脚本具有执行权限
7. 构建日志清晰易懂
